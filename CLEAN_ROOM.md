# Clean-Room Implementation Protocol (Educational)

## Goal
Implement a screenshot/annotation tool with similar capabilities, while reducing risk of creating a derivative copy.

## Rules used for this codebase

1. **No source ingestion**
   - Do not read/copy Flameshot source files into this project.
2. **Behavior-level specification only**
   - Use public descriptions/screenshots/docs to define user-facing behavior.
3. **Independent naming and structure**
   - Different module names, class names, and architecture.
4. **Original implementation**
   - New code, comments, and UI copy written from scratch.
5. **Traceability**
   - Keep this protocol in repo so intent/process is auditable.

## Still important

- This protocol lowers risk but is **not a legal guarantee**.
- For distribution/commercial use, obtain professional legal review.
